Rigol MSO1104 Oscilloscope GUI (Classic Control & Capture)
  แอปพลิเคชันเดสก์ท็อปที่พัฒนาด้วยภาษา Python และ Tkinter สำหรับควบคุมและดึงภาพหน้าจอจากออสซิลโลสโคป Rigol MSO1104 (และรุ่นอื่น ๆ ในตระกูล MSO/DS1000Z series) ผ่านโปรโตคอล SCPI และ         อินเตอร์เฟส VISA ตัวโปรแกรมมาพร้อมกับ   Simulation Mode ในตัว ทำให้สามารถเปิดทดลองใช้งานหน้าตา UI ได้ทันทีแม้จะไม่ได้เชื่อมต่ออยู่กับเครื่องมือวัดจริง

โปรแกรมที่ต้องติดตั้ง (Dependencies & Requirements)
  โปรแกรมนี้ทำงานบน Python 3.8 ขึ้นไป โดยจำเป็นต้องติดตั้งแพ็กเกจเสริมดังต่อไปนี้:
  PyVISA: ไลบรารีหลักในการสื่อสารกับเครื่องมือวัดผ่านอินเตอร์เฟส VISA
  PyVISA-py (แนะนำสำหรับผู้ที่ไม่มี NI-VISA): ตัวขับเคลื่อน (Backend) สำหรับจัดการพอร์ต USB/LAN แบบ Open-source
  Pillow (PIL): ใช้สำหรับจัดการ ดัดแปลง และแสดงผลภาพหน้าจอที่ดึงมาจากสโคปบนหน้าต่าง Tkinter

วิธีใช้งาน
  1.ต่อสาย USB หรือ LAN จากคอมพิวเตอร์เข้ากับเครื่อง Rigol MSO1104
  2.กดปุ่ม Refresh เพื่อค้นหาตำแหน่งของอุปกรณ์ (VISA Resource)
  3.เลือกพอร์ตเชื่อมต่อที่ปรากฏในช่อง Combobox (เช่น USB0::0x1AB1::0x04CE::DS1ZA...::INSTR)
  4.กดปุ่ม Connect เมื่อเชื่อมต่อสำเร็จ แถบสถานะด้านล่างจะแสดงหมายเลขเครื่อง (*IDN?) ของสโคปขึ้นมา
  
  การควบคุมช่องสัญญาณและสเกล (Control Panel)
  Channel View Control: ติ๊กเลือกเปิด/ปิดแชนเนล CH1 - CH4 เพื่อสั่งให้เส้นสัญญาณแสดงผลหรือซ่อนตัวบนสโคป
  Scale Control:
  Vertical (Volt/Div): เลือกแชนเนลที่ต้องการปรับ จากนั้นกดปุ่ม + / - หรือเลือกค่าพรีเซ็ตในกล่อง Combobox เพื่อปรับความกว้างแรงดัน (หน่วย Volt)
  Horizontal (Time/Div): กดปุ่ม + / - หรือเลือกสเกลเวลาที่ต้องการปรับความละเอียดของแนวแกนเวลา (หน่วยวินาที)

  การดึงภาพหน้าจอและบันทึกภาพ (Screen Capture)
  1.กดปุ่ม Capture โปรแกรมจะทำงานเบื้องหลัง (Background Thread) เพื่อดึงข้อมูลภาพหน้าจอ PNG จากเครื่องจริง 
  ซึ่งขั้นตอนนี้ จะใช้เวลาประมาณ 1-3 วินาที (ขึ้นอยู่กับสายเชื่อมต่อ)
  2.เมื่อภาพแสดงขึ้นมาแล้ว คุณสามารถกดปุ่ม Save Image... เพื่อเลือกพื้นที่บนคอมพิวเตอร์ในการบันทึกภาพเซฟเป็นไฟล์ .png หรือ .bmp ได้ทันที

  การส่งคำสั่ง SCPI อิสระ (Direct SCPI Terminal)
  1.พิมพ์คำสั่งมาตรฐานที่คุณต้องการทดสอบลงในกรอบ SCPI Command (เช่น :ACQuire:MDEPth? เพื่อเช็ค Memory Depth) แล้วกด Enter หรือคลิกปุ่ม Send
  2.การตอบกลับ (Response) และสถานะการทำงานทั้งหมดจะแสดงบนกรอบ Response Log ด้านล่างสุด พร้อมประทับเวลา (Timestamp) กำกับอย่างเป็นระบบ

#Brief Explanation of the GUI Design
  การออกแบบหน้าต่างโปรแกรม (GUI) มุ่งเน้นไปที่ ความง่ายในการใช้งาน (User-Friendly) และ ความเป็นระเบียบ (Structured Layout) โดยจัดสัดส่วนแบบเป็นบล็อกเพื่อให้อ่านง่ายและใช้งานได้คล่องตัว
  Logical Partitioning (การแบ่งสัดส่วนตามการทำงาน): ใช้ ttk.LabelFrame ล้อมรอบแต่ละกลุ่มคำสั่งแยกจากกันอย่างชัดเจน ช่วยให้สายตาของผู้ใช้สามารถแยกแยะกลุ่มฟังก์ชันการเชื่อมต่อ (Connection), ส่วนควบคุมการแสดงผล (Control Panels), และส่วนประมวลผลภาพ (Capture & Log) ได้ทันที
  Color-Coded Feedback: การใช้สีข้อความแสดงผลของช่องสัญญาณ CH1 - CH4 ให้ตรงกับสีมาตรฐานบนหน้าจอเครื่องสโคปจริงของ Rigol ช่วยลดความผิดพลาดในการเลือกปรับสเกลของแชนเนล
  Non-Blocking User Interface: ออกแบบการทำงานผ่านระบบ Multi-Threading โดยเฉพาะขั้นตอนการดึงภาพหน้าจอ (Capture) ซึ่งปกติจะใช้เวลาดาวน์โหลดข้อมูลค่อนข้างนาน ทำให้หน้าจอ GUI ไม่เกิดอาการค้าง (Not Responding) ระหว่างดึงภาพ
  Real-time Log Terminal: มีกล่องรับข้อมูล Log อยู่ด้านล่างสุด คอยประทับเวลา (Timestamp) ทุกคำสั่งที่ส่งไปและทุกข้อความที่ตอบกลับมา เพื่อช่วยให้ผู้ใช้ตรวจสอบสถานะการทำงานได้อย่างมั่นใจ

#Brief Explanation of the OOP Class Structure
  1. Model Layer (คุมการจัดการอุปกรณ์และคำสั่ง SCPI)
  -RigolInstrument เป็นคลาสหลักที่คุม Logic ในการติดต่อสื่อสารกับอุปกรณ์จริง ผ่านคำสั่ง SCPI บนพอร์ต VISA เช่น การ    อ่าน-เขียนข้อมูลดิบ, การดึงภาพหน้าจอในรูปแบบ Binary Block, การดึงสเกล และสถานะช่องสัญญาณ
  -SimulatedInstrument (Inheritance & Polymorphism): คลาสย่อยที่สืบทอดคุณสมบัติมาจาก RigolInstrument แต่    ทำหน้าที่ตัดกลไกการเชื่อมต่อฮาร์ดแวร์จริงออกไป แล้วแทนที่ด้วยการสวมรอยตอบกลับข้อมูลแบบสุ่ม (Mocking) และการสร้างภาพหน้าจอจำลองด้วยไลบรารี Pillow แทน เพื่อให้โปรแกรมยังสามารถรันเล่นและสาธิตหน้าตา UI ได้แม้ไม่มีอุปกรณ์จริง

  2. View Layer (คุมการแสดงผล UI แต่ละส่วนแยกอิสระ) การแบ่งหน้าต่าง UI ออกเป็นคลาสย่อยที่สืบทอดมาจากคลาสพื้นฐานของ Tkinter ช่วยลดความซับซ้อนของโค้ดลงอย่างมาก:
  ConnectionFrame: ควบคุมคอมโบและปุ่มสำหรับการเชื่อมต่อ VISA Resource
  CommandFrame: จัดการส่งคำสั่ง SCPI อิสระและปุ่มลัด
  ChannelControlFrame: สวิตช์ Checkbox เปิด/ปิด แต่ละแชนเนล
  ScaleControlFrame: จัดการปุ่มกดปรับระดับ Volt/Div และ Time/Div และค่าพรีเซ็ต
  CaptureFrame: พื้นที่สำหรับการดาวน์โหลด แปลงไฟล์ภาพ และนำภาพจำลองหรือภาพสโคปจริงมาแสดงผล
  LogFrame: กล่องข้อความแบบเลื่อนได้ (ScrolledText) สำหรับพิมพ์รายละเอียดการประมวลผล

  3.Controller Layer (ผู้ประสานงานและจัดการ Event)
  OscilloscopeApp: เป็นคลาสหลักที่ทำหน้าที่สืบทอดคุณสมบัติมาจาก tk.Tk และเป็นตัวกลางเชื่อมระหว่าง View และ Model เมื่อผู้ใช้กดปุ่มที่ View คลาส Controller จะดักจับ Event นั้น แล้ววิ่งไปเรียกใช้ฟังก์ชันใน Model เพื่อสั่งการสโคป จากนั้นจึงนำผลลัพธ์ที่ได้ย้อนกลับมาส่งอัปเดตหน้าตา UI ของ View ให้ถูกต้องตามสถานะจริง
